
📜 Founder Rules
Let’s actually write them down. These will be our “constitution.”
Rule #1
No feature gets built unless we know exactly why it exists.
Every feature must answer:
What problem does it solve?
Who asked for it?
How will we know it worked?
If we can’t answer those, it doesn’t get built.

Rule #2
If our mothers can’t explain what the app does in one sentence, it’s too complicated.
Simple beats clever.

Rule #3
Fall in love with the problem. Be ruthless about the solution.
The solution can change.
The problem stays.

Rule #4
Validation before implementation.
No feature because it’s “cool.”
Build because users need it.

Rule #5
Data wins every argument.
If I say one thing.
You say another.
Users say something else.
Users win.
No ego.

Rule #6
Trust is our first feature.
Not chat.
Not AI.
Not payments.
If users don’t trust us, nothing else matters.

Rule #7
Build the smallest thing that teaches us the most.
Not the biggest thing.
Not the prettiest thing.
The fastest way to learn.

Rule #8
Every week, ask: “What did we learn?” instead of “What did we build?”
Building is easy.
Learning is valuable.

I think this idea is going to change a lot over the next few weeks.
Maybe 20%.
Maybe 80%.

If six months from now someone says,
“This isn’t the idea you originally had.”

“Exactly. The users taught us something better.”
